play2prep Content Integration API

Academic content publishers who want global distribution through the play2prep Android and iOS app can integrate with play2prep via the Hosted Content Publisher API. Publishers can gain online and offline engagement in multiple languages, and integrate adaptive learning platforms with play2prep so that they can drive play2prep's Daily Training schedule, and all other learning formats, off of their personalized adaptive learning intelligence. To integrate content and adaptive learning, the Content Publisher hosts a content server implementing this API. The play2prep server calls to the hosted content server, to fetch new questions for students, and report their answers back to the publisher.

Question Content

The publisher provides the full question content, including text, images, answer key, explanation, and metadata. A Question Content Object is a JSON object containing HTML fragments, supporting inline images and inline ASCII Math, LaTeX, and MathML. All Question Content Objects are fully self-contained, and may not reference any external assets. Play2prep caches question content forever, and will request the contents of any question it doesn't have in the cache. This process provides enough freedom for the content publisher to add new questions and categories on the fly, without coordinating with play2prep.

Question Schedules

The Content Publisher builds personalized schedules for each student, in order to provide variety, and target areas that students need to work on. The logic that goes into building the schedule is up to the content publisher. It could be as simple as a random sequence of questions, it could provide an even variety of question across a diverse spread of categories, or it could employ adaptive learning intelligence to conduct assessments and deploy a personalized training schedule. Play2prep needs to fetch a lengthy schedule up front, for offline training, and it will refetch the schedule whenever it submits answers, so that the adaptive learning platform can steer the student as soon as it collects any new information about them. A Question Schedule is a sequence of Question Packs. Each pack is a sequence of questions that may be presented to the student as a Daily Training Pack, Practice Test, Daily Countdown Game, or as question options in the two player Ask & Answer game. Play2prep may fetch many schedules before it submits any answers for that student, so the Publisher may want to keep track of the questions it's already issued in recent schedules.

/schedules/{studentId} get

get

get /schedules/{studentId}

  • Request
  • Response

URI Parameters

  • studentId: required (string)

Query Parameters

  • Pack Count: required (integer)

    The number of packs to return in the schedule. The nature of the packs could be randomized, or they could come from some adaptive learning intelligence.

  • Pack Size: (integer)

    The size of each pack in the schedule.

  • Test Type Filter: required (string)

    Test type code, like "SAT", "ACT", "SAT2_BIO", "SAT2_PHYS", etc. The play2prep server should never request a test type the publisher doesn't provide.

  • Difficulty Filter: (string)

    Comma separated list of difficulty ids, denoting a whitelist/blacklist of difficulty levels. Ids starting with a ! denoted a blacklisted level. For example, to include 1 and 2 and exclude 3, 4, and 5 - "difficulty=1,2,!3,!4,!5", For example, to include all difficulty levels except 5 - "difficulty=!5"

  • Category Filter: (string)

    Comma separated list of category names denoting a whitelist/blacklist of categories. Names starting with a ! denote a blacklisted name. For example, to include only Math and Physics - "categories=Math,Physics". For example, for a hierarchical symbology like category names, it may be necessary to mix whitelist and blacklist items. For example, to include Math but exclude sub category Geometry - "categories=Math,!Geometry". Or, to include all categories except Animal Biology - "categories=!Animal%20Biology".

  • Answer Format Filter: (string)

    Comma separated list of answer format codes, denoting a whitelist/blacklist of answer formats. While the publisher may have questions for a variety of interactive formats, play2prep may not support all of them yet, so the play2prep server needs to list out all the formats it supports. For example, to get only multiple choice and student produced response formats - "answerFormats=MC,SPR". All the currently supported formats are - MC (Multiple Choice), SPR (Student Produced Response), and GRIDIN (SAT Math Grid-In).

Answers

Play2prep sends all answers to the content publisher, so that they can track students progress and do their own bookkeeping. Play2prep used its cached copy of the answer key to grade and score the questions already, so it doesn't need any response from the publisher about correctness or scoring.
